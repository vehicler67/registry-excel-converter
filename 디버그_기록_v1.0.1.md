## 버그 #1: 기본정보 추출 실패 (고유번호·소재지·부동산종류)

**증상**: 전체요약 시트에서 열람일시만 추출되고 나머지 3개 필드는 "(추출 실패)"

**원인 1** — 정규식이 `고유번호 : 2849-...` 형식만 기대했으나 실제 OCR 출력은 `고유번호2849-2018-019318` (구분자 없이 붙어나옴)

**원인 2** — OCR 결과에 "소재지:" 라벨 자체가 없고 `[토지]경기도파주시...` 형식

**원인 3** — parse_sections()가 첫 번째 【 섹션 이전 라인(고유번호, 소재지)을 버리고 있었음

**수정**:
- `_extract_basic_info()`: 구분자 없는 직접 연결 패턴 추가, [토지] 태그 fallback
- `parse_sections()`: 첫 섹션 시작을 0으로 보정 (actual_start = 0 if idx == 0)

## 버그 #2: 표제부 "데이터 없음" (섹션명 불일치)

**증상**: 전체요약 시트 하단에 `[[표제부]] — 데이터 없음` 표시

**원인**: OCR 결과가 `【표제부】(토지의표시)` 인데, 렌더러는 `【표제부】`로만 조회 → 키 불일치

**수정**: `parse_sections()`에 섹션명 정규화 로직 추가
- `【표제부】(토지의표시)` → `【표제부】`
- `【매매목록】` → `【매매목록】`
- 모든 섹션명을 표준 형식으로 통일

---

## 버그 #3: 갑구/을구 섹션 미감지

**증상**: 갑구 5행, 을구 빈 시트

**원인 1**: OCR이 `【을구】`를 `【을 _ 구】`로 깨뜨려 키워드 매칭 실패
**원인 2**: 갑구 헤더가 PDF에서 완전히 누락되는 케이스 존재

**수정**:
- 공백/특수문자 제거 후 부분 매칭 (`re.sub(r'\s+', '', stripped)`)
- 갑구 누락 시 "순위번호 | 등기목적" 컬럼헤더로 보조 감지

---

## 버그 #4: detect_table_regions IndexError

**증상**: `IndexError: tuple index out of range` (그리기 아이템 구조 불일치)

**원인**: PyMuPDF drawing items에 'l'(line) 외 're'(rect) 등 다른 op 타입 존재

**수정**: item 길이 검증 + op 타입별 분기 처리 (rect → 상하좌우 4선 분해)
